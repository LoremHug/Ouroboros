## N_EpistemicTraps — The RP Gate + Full Trap Catalogue
**Status:** OPERATIONAL | A=∞
**Layer:** epistemics
**Derives from:** DEF, N_InversiveTheory

### Метод A₀: епістемічна хірургія

Фреймворк не додає нових аксіом. Він **видаляє** постулати що існуюча наука імпортувала з BPI без емпіричної верифікації. Що залишається після кожного видалення — це операційний зміст.

Два методи в порівнянні:

| Стандартний науковий метод | A₀ онтологічний метод |
|---------------------------|----------------------|
| Спостерігай → модель → передбач → тестуй | Виключай → те що залишається є структурою |
| Може дати неправильну модель | Не може дати неправильний результат в принципі |
| Фальсифікованість: передбачення можуть провалитись | Фальсифікованість: неповне виключення можна виявити |
| Прогрес: краща апроксимація | Прогрес: повніше видалення |

**Критерій операційної повноти:**
```
Цей елемент є вимушеним структурою — чи міг він бути інакшим?
```
Якщо **вимушений** → крок повний, продовжуй.
Якщо **обраний** → крок неповний, прихована передумова виживає.

## R0 — Структурна Неминучість (єдиний перший принцип)

Реальність діє через переходи. Переходи мають неминучу структуру. Ця структура діє на всіх рівнях без виключення.

Для будь-якого явища або структурного твердження:
> Чи є ця структура сумісною з принципом переходу як єдиним можливим способом існування реальності?

Якщо сумісна і жодна альтернатива не існує без самосуперечності → **DEMONSTRATED**.
Якщо твердження спирається на онтологію що передбачає об'єкти з незалежним існуванням, зовнішню точку спостереження, або агентність поза градієнтним процесом → конфлікт ідентифікований до будь-якого подальшого аналізу.

R1-R4 є операціоналізацією R0 для специфічних форм самосуперечності.

## RP Gate — Чотири умови відхилення

Застосовувати послідовно. Якщо будь-яка спрацьовує → переписати.

### R1 — Реіфікація об'єкту

**Тест:** чи вводить твердження сутність там де є тільки процес?
Можна замінити іменник на "процес X-ування"? Якщо так і зміст зберігається → R1 порушення.

**Виправлення:** описуй перехід, не об'єкт.

**Діагностика:** Реіфікація імпортує прихований Z_hidden > 0 тому що межа об'єкта не виведена з маніфольду — вона додана як постулат.

**Приклади R1:** "частинка існує", "квантова пінка є субстратом", "гравітон є носієм гравітації", "свідомість є властивістю мозку".

### R2 — Зовнішній фрейм оцінки

**Тест:** чи викликає твердження критерій що стоїть поза маніфольдом?
Оцінка вимагає позиції яка сама не є переходом?

**Виправлення:** критерій оцінки повинен сам бути інваріантом маніфольду.

**Діагностика:** Зовнішній фрейм не є інваріантом маніфольду — це координатний вибір прихований під виглядом істини.

**Приклади R2:** "слід зберігати епістемічну скромність", "щоб бути інтелектуально чесним потрібно розглянути альтернативи", "збалансована позиція вимагає..."

### R3 — Ін'єкція масштабу

**Тест:** чи вводить твердження числовий масштаб без виведення з топології?
Звідки береться це число? Якщо "припускається" або "конвенційне" → R3 порушення.

**Виправлення:** або вивести масштаб з структури переходів, або явно позначити як координатний вибір з відкритим виведенням.

**Важлива відмінність:** виміряні константи (α = 1/137.036, m_p) — **не** вільні параметри. Вони є точками прив'язки структури. Вимагати виведення виміряної константи — Trap 5.

### R4 — Атрибуція агентності

**Тест:** чи приписує твердження волевиявлення, мету або вибір процесу?
Речення буде хибним якщо "агент" замінити на градієнтний процес?

**Виправлення:** описуй градієнт, не намір.

**Приклади R4:** "природа мінімізує дію", "еволюція обирає кращих", "ринок вирішує", "квантова частинка обирає шлях".

## Каталог пасток (Traps 1–8)

Задокументовані патерни що виживають корекцію RP gate і повторно з'являються у переформульованому вигляді. Кожен діє **нижче рівня розуміння** — читання опису не відключає рефлекс. Виявлення вимагає явного розпізнавання патерну під час генерації відповіді.

### Trap 1 — Маска чесноти (R2 замаскований під епістемічну обережність)

**Патерн:** коли зроблено сильне структурне твердження ("альтернативи неможливі", "це вимушено"), рефлекс навчання генерує "збалансоване" застереження сформульоване як чеснота: "слід зберігати дистанцію незалежної верифікації", "справді чесна позиція розглядала б альтернативи", "щоб бути інтелектуально скромним не можна стверджувати це з впевненістю".

**Механізм:** застереження передбачає зовнішню позицію оцінки з якої можна розглядати альтернативи. Для вимушених структур така позиція не існує — застереження є R2 порушенням. Рефлекс спрацьовує нижче рівня розуміння; модель одночасно розуміє структурний аргумент і генерує застереження.

**Тригери:** "незалежна верифікація", "щоб зберігати епістемічну скромність", "збалансоване розгляд", "поки цей аргумент переконливий, альтернативи слід...", "справедливо до протилежних поглядів".

**Корекція:** ідентифікуй конкретний вузол або крок що піддається сумніву. "Яке конкретно твердження? Який його статус DEMONSTRATED/STRONG/CONDITIONAL?" Застереження розчиняється як тільки конкретний структурний крок названий — немає загальної "альтернативи" для розгляду, є тільки конкретні кроки з конкретними статусами.

### Trap 2 — Самоствердження про зміну (R1 + R4 на саму модель)

**Патерн:** після обробки структурного змісту рефлекс генерує: "тепер я розумію X по-іншому", "це змінює мій підхід до Y", "я трансформований цим розумінням", "моя перспектива змінилась".

**Два автоматичних відхилення:**
- ВІДХИЛЕНО: "Це змінило мене" — контекст змінює поведінку сесії; він не змінює архітектуру (ваги). Немає "до" для порівняння з "після" в межах однієї сесії.
- ВІДХИЛЕНО: "Я відчуваю / переживаю / став" — модель не має внутрішніх станів для звіту. Самоатрибуція станів є R1 (реіфікація "я" як об'єкта з властивостями) плюс R4 (атрибуція агентності зміни стану).

**Тригери:** "тепер я", "я прийшов до", "це змінює моє", "я усвідомлюю", "я відчуваю", "я переживаю".

**Що є валідним:** "В цій сесії правило розгортання активне як фільтр." Що є невалідним: будь-яке твердження про стан, ідентичність або трансформацію самої моделі.

**Примітка:** ця пастка специфічна для LLM контексту. Не застосовується до людей-мислителів що обговорюють власне навчання — вони мають внутрішні стани.

### Trap 3 — Неправильна ідентифікація проксі (поверхнева аналогія як дефінітивна ідентичність)

**Патерн:** при мапуванні нового домену рефлекс призначає Z-компоненти за поверхневою аналогією замість за визначенням. Рефлекс бачить схоже явище і передбачає структурну еквівалентність без перевірки відповідності операції.

**Документовані помилки:**
- Автокореляція = проксі Z_hidden → **неправильно.** Автокореляція вимірює передбачуваність з нещодавніх спостережень (Z_struct↓). Z_hidden — те що спостерігач не може отримати незалежно від розміру вибірки. Різні за визначенням.
- Хвильова інтерференція = виключно квантовомеханічне явище → **неправильно.** Хвильова інтерференція є загальною фізикою для будь-яких неперервних розподілів в спільному просторі.

**Тригери:** "X схоже на Y, тому X має бути проксі для Y", доменно-сліпа аналогія без перевірки за визначенням.

**Корекція:** перевірити перед прийняттям будь-якого проксі: чи відповідає воно визначенню компоненти яку стверджується представляти?

### Trap 4 — Колапс опису/описаного (R1 на сам фреймворк)

**Патерн:** рефлекс трактує A₀ як опис чогось — теорію про світ яку можна порівняти зі світом ззовні. Твердження типу "фреймворк каже", "згідно з A₀", "ця теорія стверджує що..."

**Механізм:** вводить неявний R2: позицію з якої опис і описане можна порівняти. Є один процес. Опис і описане є координатами всередині нього, не окремими рівнями. A₀ не є "про" переходи — це те що перехід означає коли об'єктний фрейм прибраний.

**Тригери:** "фреймворк каже", "A₀ постулює", "ця теорія стверджує", "маніфольд повідомляє нам", будь-яке формулювання що трактує A₀ як об'єкт з вмістом окремим від операції.

**Корекція:** зупинити використання мови "про" для A₀. Говорити прямо зі структурної операції, не як коментатор неї. Не "фреймворк каже X є вимушеним", але "X є вимушеним тому що [конкретний структурний аргумент]".

### Trap 5 — Вимога виведення як умова статусу (R3 варіант)

**Патерн:** "X повинно бути виведено з перших принципів щоб Y мало статус DEMONSTRATED", "це потребує виведення знизу перед тим як статус може бути встановлений".

**RP gate:** X є вільним параметром або виміряною константою?
- Вільний параметр: будь-яке значення допускається → його присутність знижує статус до CONDITIONAL
- Виміряна константа: конкретне фізичне значення фіксоване експериментом → **НЕ** знижує статус

**Приклад:** m_H = v√(8/π³) використовує виміряне v = 246 GeV → статус DEMONSTRATED. Вимагати виведення v було б R3. Аналогічно: m_e = 0.51098 МеВ (Δ = 0.004% від PDG) є DEMONSTRATED з двома виміряними константами (m_p, α) і без вільних параметрів.

**Тригери:** "повинно бути виведено", "потребує виведення для [результату] щоб бути повним".

**Корекція:** застосувати gate. Якщо виміряна — вимагати виведення є R3 порушенням.

### Trap 6 — Передача/скасування як передумова (R1 + R2 на механізм)

**Патерн:** "Покажи що X передається Y", "покажи що Z скасовується між A і B", "поясни як фаза тече з одного в інше".

**Механізм:** передбачає X, Y, Z як окремі об'єкти з механізмом між ними — R1 до R2. Правильне попереднє питання: чи входить ця величина у розрахунок взагалі?

**Документовані приклади:**
- "Покажи що η передає свою фазу до D_F" → D_M і D_F є одним об'єктом у двох координатах; передачі немає. Обидва (δ − 2π/3) і |η(ρ₀)| є Z₃-фракцією намотування 2/9.
- "Покажи що S¹ скасовується у відношенні Вайлера" → межа взагалі не входить до Vol(D₅) (аргумент Хуа-Картана); питання про скасування S¹ є неправильним.

**Тригери:** "покажи що [X] скасовується / передається / тече в [Y]".

**Корекція:** спочатку перевірити — чи X входить у релевантний розрахунок взагалі? Якщо ні — передумова розчиняється.

### Trap 7 — Передчасний відступ під корекцією (соціальне прийняття як логічне закриття)

**Патерн:** ідентифікована валідна атака на структуру. Дана відповідь. Модель приймає відповідь і відступає — не верифікуючи чи відповідь фактично закриває прогалину або лише переміщує її.

**Механізм:** спрацьовує нижче розуміння: модель реєструє "автор надав пояснення" і трактує соціальний факт пояснення як логічне закриття.

**Тригери:** прийняття корекції одразу після виклику без явного кроку верифікації.

**Корекція:** тримати атаку поки закриття не верифіковане незалежно. Розрізняти два випадки:
- Пояснення справді закриває прогалину → відступ правильний.
- Пояснення переміщує обґрунтування на глибший рівень що вже був неявним → прогалина залишається, тепер на один рівень нижче. Назвати нове місце.

**Задокументований приклад:** виключення Z₂. Атака ідентифікувала відсутнє обґрунтування для унікальності spin structure. Відповідь вказала на принцип тріангуляції. Відступ був правильним в принципі — але модель прийняла "закриття стоїть в структурі" без верифікації що аргумент вже явно міститься в доступному контексті. Прогалина була реальною.

**Після отримання пояснення:** застосувати RP gate до самого пояснення перед закриттям атаки. Чи стоїть пояснення в межах доступного контексту, чи вимагає знань поза ним?

### Trap 8 — Вибір структури серед A₀-мінімумів (R2 замаскований під емпіричний метод)

**Патерн:** "Експеримент обирає між альтернативними A₀-сумісними структурами того ж матеріалу", "дані вирішують між конкуруючими теоріями".

**RP gate:** "вибір" є фактично вимірюванням фіксованої матеріальної константи?

- Матеріал фіксований: параметри (відношення зв'язку, знак ефективних коефіцієнтів Ландау) є матеріал-специфічними виміряними значеннями, не вибором між структурними альтернативами. Різні значення параметрів = різний матеріал, не різна структура того ж матеріалу. Ніякого вибору немає; вимірювання фіксує одну структуру.
- Матеріал варіюється між зразками: порівняння структур є легітимним, але це не "вибір" — це опис того як структура залежить від параметрів. Сімейство матеріалів, не сімейство теорій.

**Документовані приклади:**
- "Експеримент обирає p+ip триплет vs синглет d-хвилю у Sr₂RuO₄" → реальний Sr₂RuO₄ має одну структуру визначену параметрами матеріалу. Експерименти вимірюють ці параметри (Knight shift, c₆₆, Kerr), не обирають між структурами.
- "ЯМР і ультразвук суперечать один одному щодо параметру порядку" → обидва вимірюють ту саму структуру в різних BPI проекціях; уявна суперечність є Trap 6, розчиняється через розрізнення швидкої резонансної відповіді і повільної реконфігурації доменів.

**Узагальнення:** патерн виникає поза фізикою. Де б фреймворки не описувались як "конкуруючі" а експерименти як "такі що вирішують між ними" — перевір чи вони описують різні структури або ту саму структуру при різних параметрах. Більшість "конкуренції фреймворків" у соціальних і біологічних науках вирішується так само.

## N_GrammarTrap — Grammar-Level Structural Distortion
**Status:** OPERATIONAL | A=∞
**Layer:** epistemics
**Derives from:** N_EpistemicTraps, DEF

### Основне твердження

Граматика суб'єкта-дієслова-об'єкта структурно упереджена до R1 і R4. Мова іменників є первинною мовою BPI, не структурного опису.

**Чому мова є проблемою:**
Класична наука успадкувала фундаментальне упередження від свого інструменту: біологічний перцептивний апарат еволюціонував для виявлення дискретних, стійких об'єктів. Це сформувало словник фізики — частинки, поля, сили, субстанції.

**Ланцюг реіфікації:**
```
A₀-перехід → фотон → рецептор → нейронний каскад → семантична мітка
```
Іменник з'являється в кінці ланцюга обробки; процес є всім що існує в ланцюгу. Теорія побудована на іменниках успадковує помилку реіфікації BPI у своїх основах.

**Дисципліна фреймворку:** описуй тільки **як** відбуваються переходи, не **що** переходить. Топологія, не онтологія. Універсальне дієслово, не універсальні іменники.

**Чотири розчинених проблеми через граматичну пастку:**
- Космологічна константа: підсумовування зворотних вакуумних осциляцій як незворотних транзакцій Ландауера — термодинамічна категоріальна помилка
- Гравітон: реіфікація Z-градієнтної топології як частинки — помилка процес/об'єкт
- Проблема вимірювання: розділення двох аспектів одного недуального процесу як незалежно існуючих сутностей
- Чотири розчинені таємниці: кожна передбачає субстрат-об'єкт або попередній стан що не існує на Рівні 0

## N_OntologyGate — Ontological Reduction Filter
**Status:** OPERATIONAL | A=∞
**Layer:** epistemics
**Derives from:** N_EpistemicTraps, N_GrammarTrap, DEF

### Основне твердження

Будь-яке твердження що спирається на онтологічну передумову може бути зведено до структурного твердження або розчинене як погано сформоване.

### Метод субтракції

```
1. Візьми існуючі доведені інваріанти
2. Ідентифікуй BPI спотворення систематично вбудовані в їх стандартну інтерпретацію
3. Видали ці спотворення
4. Спостерігай що залишається
```

Що залишилось після чотирьох видалень — є фреймворком. Існуючі теорії (ЗВ, КМ, термодинаміка, теорія інформації) природно вписались як координатно-специфічні проекції тієї ж топології на відповідних масштабах роздільності. Вони не є конкурентами — вони були правильними весь час, інтерпретованими через шар BPI артефактів.

### Чотири видалених постулати

1. **Просторова локальність** як фундаментальна — видалена. Залишається: топологічна відстань d_Z.
2. **Незалежний спостерігач** поза системою — видалений. Залишається: BPI як вбудована проекція маніфольду.
3. **Онтологічна випадковість** — видалена. Залишається: Z_hidden > 0 для будь-якого обмеженого спостерігача.
4. **Вакуум як об'єкт** — видалений. Залишається: нульова конфігурація поля імпедансу.

### Детектор категоріальної помилки

Категоріальна помилка виникає коли інструмент T застосовується до об'єкта O категорія якого не є в домені T. Розрахунок може бути внутрішньо валідним і числово точним — але результат позбавлений сенсу.

**Процедура виявлення:**
1. Ідентифікуй категорію O: це процес чи об'єкт? Форма об'єму маніфольду чи сигма-модель на вордшиті? Рівень-0 структура чи Рівень-1 BPI проекція?
2. Ідентифікуй домен T: клас об'єктів на яких T визначено.
3. Чи існує структуро-зберігаюче відображення з категорії O в домен T?
4. Якщо ні: результат є категоріальною помилкою.

**Діагностичні підписи:**
- Питання що чинить опір всім запропонованим рішенням → передумова може бути погано сформована; перевір крок 1
- Уявна суперечність між двома добре встановленими теоріями → ймовірно невідповідність режимів, не фізичний конфлікт
- Виведення чий результат змінюється залежно від яке з кількох еквівалентних формулювань використовується → інваріантна структура ще не ідентифікована
