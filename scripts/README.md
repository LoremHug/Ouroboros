# Ouroboros toolkit

Local Kuzu-backed structural graph + map generator. Single venv, no external services.

## Setup (one-time)

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

## Build / rebuild the database

```bash
.venv/bin/python -m scripts.migrate         # parse manifold_graph.txt → manifold.kuzu
.venv/bin/python -m scripts.enrich_from_tex # add .tex sections + DESCRIBED_BY links
.venv/bin/python -m scripts.generate_map    # write web/index.html
```

`migrate` drops the DB and rebuilds. `enrich_from_tex` re-creates the Section table.
`generate_map` is read-only; run it any time to refresh `web/index.html`.

## Open the map

```
file:///home/user/Ouroboros/web/index.html
```

Features:
- click node/edge → side panel with full content
- click `.tex section` row in panel → section text
- search box → filter / press Enter to focus
- legend layer → click to filter (multi-select)
- scroll = zoom, drag = pan, esc = close panel

## CRUD

```bash
# Add or update a node
.venv/bin/python -m scripts.add_node N_NewNode \
  --layer observers --status STRONG --anchors 3 \
  --title "Some Forced Identification" \
  --summary "One-line claim" \
  --content "Full structural argument..."

# Add an edge with full justification
.venv/bin/python -m scripts.add_edge N_BPIEngagement N_Shannon \
  --label shannon_capacity_is_BPI_constraint \
  --status D \
  --justification "K(O)<K(F) and Shannon capacity are one structure" \
  --why-forced "Alternative requires two independent info-limit mechanisms"
```

After editing run `generate_map` again to refresh the HTML.

## Query

```bash
# Raw Cypher
.venv/bin/python -m scripts.query 'MATCH (n:Node) WHERE n.layer = "core" RETURN n.id, n.status'

# Convenience flags
.venv/bin/python -m scripts.query --neighbors N_BPIEngagement --depth 2
.venv/bin/python -m scripts.query --status CONDITIONAL
.venv/bin/python -m scripts.query --orphans
```

## Find structural gaps

```bash
.venv/bin/python -m scripts.find_gaps                  # all checks
.venv/bin/python -m scripts.find_gaps --jaccard 0.4    # candidate missing edges
.venv/bin/python -m scripts.find_gaps --stubs          # nodes lacking content
.venv/bin/python -m scripts.find_gaps --unjustified    # edges missing why_forced
```

`--jaccard` looks for node pairs sharing many neighbors but no direct edge — these
are candidate forced-identifications worth investigating.

## File layout

```
scripts/
  db.py              — Kuzu connection + schema bootstrap
  migrate.py         — manifold_graph.txt → manifold.kuzu
  enrich_from_tex.py — THE_IMPEDANCE_MANIFOLD_v3_6.tex → Section nodes + links
  generate_map.py    — Kuzu → web/index.html (self-contained)
  add_node.py        — upsert a node
  add_edge.py        — create or update an edge
  query.py           — Cypher / convenience queries
  find_gaps.py       — heuristic structural gap detection
schema/
  models.py          — Pydantic Node/Edge, Status/Layer enums, layer map
manifold.kuzu/       — Kuzu DB (gitignored, regenerated by migrate)
web/
  index.html           — generated visualisation
```
